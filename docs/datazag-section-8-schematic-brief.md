# Design Brief: Homepage Section 8 Schematic

**For:** [designer who produced the ESP schematic]

**Reference image:** `ESP_schematic.jpeg` (the existing ESP partner page hero diagram)

**Purpose:** Sibling diagram for homepage Section 8 ("Where we fit, and why it holds up"). Same visual template as the ESP version, with content adapted to be universal rather than ESP-specific.

**Page context:** Homepage. Section 8 sits after the products grid and the channel-segment routing, before the proof stats and final CTA. The section's job is to position Datazag as a layer alongside the buyer's existing tools, and to credential the engine that makes that position defensible. The diagram needs to do both jobs in one visual.

## What to keep from the ESP version (unchanged)

**Overall composition.** The horizontal left-to-right flow: Datazag Graph on the left, real-time signals feed arrow in the centre, consuming stack on the right, outcome icon on the far right. Same proportions, same panel structure.

**Colour palette.** The teal / purple / dark navy with point-light constellation background. Don't change the brand palette. Visual cohesion with the ESP version is the entire point.

**Left side — Datazag Graph cluster.** Keep the dark teal frame, the constellation graph with four primary nodes (corner positions) around a central labelled node, and the surrounding minor nodes with thin connecting edges. The cluster shape is correct.

**Real-time signals feed arrow.** Keep — same shape, same gradient, same "Real-Time Signals Feed" label.

**Right-side panel structure.** Keep the purple frame, three vertically stacked sub-panels, each with a labelled icon on the left and a three-item bullet list on the right.

**Outcome icon on the far right.** Keep the position and treatment.

**Schematic caption pattern.** Keep the small caption at the bottom in italic/lighter weight, in the same position. Caption text changes — see below.

## What needs to stay completely consistent with the ESP version

The look and feel of every element should be visually indistinguishable in style. Same font weights, same icon style, same node-and-edge constellation graph treatment, same gradient on the central arrow, same colour temperature. A visitor who sees both diagrams on different pages should immediately recognise they're part of the same family.

In particular: the constellation pattern on the dark teal panel, the digital-particle effect inside the central arrow, the glowing node treatment, and the typography hierarchy should all be identical to the ESP version.

## Open items for the design pass

1. **Confirm icon styles.** The ESP version's icons (cylinder, abstract-graph, globe, speedometer on the left; database-lock, server, padlock on the right) work for the homepage version if you change a few. If you'd prefer different icons for the new right-side categories (envelope-shield, DNS-filter, threat-radar), specify and propose alternatives.

2. **Confirm the centre arrow label.** "Real-Time Signals Feed" works for both versions. If a homepage-appropriate alternative is stronger (e.g., "Pre-Arrival Blocking"), discuss.

3. **Outcome icon and label.** The shift from "Authenticated, Secure Email Delivery" to a universal outcome may need icon iteration. Suggest 2-3 options for the homepage outcome icon (shield, blocked-arrow, locked-padlock, etc.) and propose.

4. **File deliverables.** The original schematic should be delivered in a vector format (SVG / PDF / Figma source) so all five planned variants can be produced efficiently from the same base file. If the original was rendered (AI image or raster), discuss whether to recreate as vector for the sibling versions.
